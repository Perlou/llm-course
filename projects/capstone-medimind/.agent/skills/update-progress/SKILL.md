# 更新开发进度技能

此技能用于更新 MediMind 健康助手项目的开发进度追踪。

## 进度文件位置

```
docs/PROGRESS_TRACKER.md
```

## 状态标记

| 图标 | 状态   | 说明     |
| ---- | ------ | -------- |
| ⬜   | 待开始 | 尚未开始 |
| 🔄   | 进行中 | 正在开发 |
| ✅   | 已完成 | 开发完成 |
| ⚠️   | 阻塞   | 遇到问题 |
| ⏭️   | 跳过   | 任务跳过 |

## 更新任务状态

### 开始任务

将状态从 `⬜ 待开始` 改为 `🔄 进行中`：

```markdown
| 1.1 | 创建 Python 虚拟环境 | 0.5h | 🔄 进行中 | | |
```

### 完成任务

将状态改为 `✅ 已完成`，填写完成日期：

```markdown
| 1.1 | 创建 Python 虚拟环境 | 0.5h | ✅ 已完成 | 2026-02-03 | |
```

## 更新统计表

完成任务后更新统计：

```markdown
| 阶段     | 总任务数 | 已完成 | 进行中 | 完成率 |
| -------- | -------- | ------ | ------ | ------ |
| Phase 1  | 10       | 5      | 1      | 50%    |
| Phase 2  | 10       | 0      | 0      | 0%     |
| **总计** | **74**   | **5**  | **1**  | **7%** |
```

## 添加每日日志

在"每日更新日志"部分添加：

```markdown
### 2026-02-04

- ✅ 完成 Guardrails 模块实现
- ✅ 完成 RAG 服务基础框架
- 🔄 正在实现 Triage Agent
- 📝 发现 Gemini API 速率限制问题，记录到风险追踪
```

## 更新风险追踪

遇到问题时记录：

```markdown
| 风险            | 状态      | 描述               | 缓解措施         |
| --------------- | --------- | ------------------ | ---------------- |
| 医学数据准确性  | ⬜ 待观察 | 需确保数据来源权威 | 仅使用官方数据源 |
| Gemini API 限制 | 🔄 处理中 | API 调用频率限制   | 增加重试机制     |
```

## 更新交付物

完成后勾选交付物：

```markdown
**本周交付物**:

- [x] src/core/guardrails.py
- [x] src/core/rag_service.py
- [ ] src/core/triage_agent.py
- [ ] tests/test_guardrails.py
```

## 阶段完成检查

每个阶段完成时：

1. ✅ 所有任务标记为完成
2. ✅ 所有交付物已勾选
3. ✅ 统计表数据更新
4. ✅ 在每日日志记录完成
5. ✅ 阶段状态改为 `✅ 已完成`
